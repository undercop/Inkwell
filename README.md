# AI Apps — Book Recommender + Blog Writer (merged)

## 1. Setup

```bash
cd ai-apps
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # macOS/Linux

pip install -r requirements.txt
```

Copy your `.env` (with `GROQ_API_KEY`, `TAVILY_API_KEY`) into the project
root — see `.env.example`.

## 2. Add your book recommender data

Copy these three files from your old book-recommender project into `data/`:

```
data/books_with_emotions.csv
data/tagged_description.txt
data/cover-not-found.jpg
```

## 3. Build the vector index once (fixes the slow startup)

```bash
python scripts/build_vector_db.py
```

This creates `data/chroma_db/` — a persisted index. Re-run this script
only if `tagged_description.txt` changes. The app will auto-detect and
load this instead of re-embedding everything on every launch.

## 4. Run the merged app

```bash
streamlit run app.py
```

You'll get a sidebar with two pages: **Book Recommender** and
**Blog Writer**. Generated blogs are saved to `data/blogs/*.md` and
listed in the "Past blogs" panel on the Blog Writer page.

## Project layout

```
app.py                        entry point — navigation only
pages/
  book_recommender.py         UI for the book recommender
  blog_writer.py              UI for the blog writer (was frontend.py)
core/
  book_recommender/
    data.py                   cached data + vector DB loading
    recommend.py              pure recommendation logic
  blog_writer/
    graph.py                  LangGraph pipeline (was backend.py)
scripts/
  build_vector_db.py          one-time index builder
data/
  books_with_emotions.csv     (you provide)
  tagged_description.txt      (you provide)
  cover-not-found.jpg         (you provide)
  chroma_db/                  generated by build_vector_db.py
  blogs/                      generated blog .md files
images/                       (optional, for blog images)
```

## Notes

- Only `app.py` calls `st.set_page_config` — don't add it to the page
  files, Streamlit only allows one call per run.
- All widget `key=` values are namespaced (`book_*`, `blog_*`) so the
  two pages never collide in `st.session_state`.
- `core/` has zero `st.*` calls in the logic modules (`recommend.py`,
  `graph.py`) — only `data.py` uses `st.cache_resource` for caching,
  which is the correct/idiomatic place for it.
